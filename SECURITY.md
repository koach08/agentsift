# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AgentSift itself, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@agentsift.dev** (or use GitHub's private vulnerability reporting feature)

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide a timeline for a fix.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |

## Scope

The following are in scope for security reports:

- Vulnerabilities in AgentSift's scanning engine
- Sandbox escape in the behavioral analysis component
- False negatives that allow known-malicious packages to pass undetected
- Supply chain risks in AgentSift's own dependencies

## Responsible Disclosure

We follow a 90-day disclosure timeline. We will:

1. Acknowledge your report within 48 hours
2. Provide an initial assessment within 7 days
3. Release a fix within 90 days (or coordinate a disclosure timeline with you)
4. Credit you in the security advisory (unless you prefer anonymity)
